# Codver — AI Coding Agent Runner

Codver automates the full lifecycle of a coding task: clone a GitHub repo, run an [PI AI agent](https://pi.dev/) inside a sandboxed Docker environment, commit the changes, and open a Pull Request. You give it a repo and a prompt — it gives you a PR.

## How it works

```
codver --repo https://github.com/owner/repo --prompt "Add unit tests for the auth module"
```

1. **Validates** host dependencies, config, API keys, and model availability
2. **Clones** the repo and creates a new branch
3. **Generates** a Docker dev environment (Dockerfile + compose) via AI
4. **Starts** the sandbox container with the coding agent
5. **Runs** the AI agent inside the container to perform the task
6. **Stops** containers and cleans up
7. **Creates a PR** with AI-generated commit messages and description, or an error report if something went wrong

## Installation

### Server (CLI)

Copies the server to `~/.codver/server/`, runs `bun install`, and adds `~/.codver/server/bin` to PATH in `.bashrc`/`.zshrc`/`.profile`/fish config.

```bash
curl -fsSL https://raw.githubusercontent.com/iambpn/codver/main/install-codver-server.sh | bash
```

Open a new terminal or run:

```bash
export PATH="$HOME/.codver/server/bin:$PATH"
```

Verify with:

```bash
codver --help
```

### Delegate Skill

The `codver-delegate` skill lets your AI coding agent delegate tasks to a remote codver server over SSH. Copies the skill to a target directory's `.agents/skills/`:

```bash
curl -fsSL https://raw.githubusercontent.com/iambpn/codver/main/install-codver-skill.sh | bash -s -- /path/to/your/project
```

The target is always treated as the project root — the skill is installed at `<target>/.agents/skills/codver-delegate`. If no target is given, the current directory is used as the project root.

### Updating

Pulls the latest server files from the repo and reinstalls dependencies, preserving your existing `codver.config.json`:

```bash
curl -fsSL https://raw.githubusercontent.com/iambpn/codver/main/update-codver-server.sh | bash
```

## Configuration

Run `codver init` to create `~/.config/codver/codver.config.json`:

```jsonc
{
  "gitUserName": "Your Name",                              // optional
  "gitUserEmail": "your.email@example.com",                // optional
  "defaultModel": "anthropic/claude-sonnet-4-20250514"     // optional
}
```

All keys are optional. You can also pass a custom config path with `--config`.

## Usage

### Run a task

```bash
codver --repo https://github.com/owner/repo --prompt "Refactor the database layer" --model anthropic/claude-sonnet-4-20250514
```

### Options

| Flag | Description |
|------|-------------|
| `--repo <url>` | GitHub repository URL (required) |
| `--prompt <text>` | Task description (mutually exclusive with `--prompt-file`) |
| `--prompt-file <path>` | Path to a file containing the task |
| `--model <provider/id>` | AI model for the coding task (e.g. `anthropic/claude-sonnet-4-20250514`) |
| `--new-branch <name>` | Branch name (auto-generated by AI if omitted) |
| `--from-branch <name>` | Base branch (defaults to the repo's default branch) |
| `--config <path>` | Custom config file path |

### Subcommands

| Command | Description |
|---------|-------------|
| `codver check` | Verify host deps, config, API keys, model, and repo access |
| `codver init` | Generate `~/.config/codver/codver.config.json` |
| `codver init --force` | Overwrite existing config |
| `codver clean` | Remove cloned repos under `~/.codver-dev/` (preserves config) |
| `codver clean --all` | Full cleanup including Docker artifacts |
| `codver clean --dry-run` | Preview what would be removed |

### Examples

```bash
# Check that everything is ready
codver check

# Run with a prompt file
codver --repo https://github.com/owner/repo --prompt-file ./task.md

# Run on a specific base branch
codver --repo https://github.com/owner/repo --prompt "Fix login bug" --from-branch develop
```

## Remote delegation

The `codver-delegate` skill lets you offload tasks to a more powerful machine. Once installed, your agent can SSH into a remote server and fire off codver tasks asynchronously.

1. Set up codver on the remote machine (same installation steps)
2. Optionally create `~/.codever-ssh` with the remote host (e.g. `user@192.168.1.100`)
3. Ask your agent: `/codver-delegate user@host add dark mode support`

The remote server runs the full pipeline in tmux (or nohup as fallback) and creates a PR when done.
